The X-Factor

EduSign: AI Video Stitching & Auto-Quiz Pipeline

The most complex engineering challenge in this project was building the automated video processing and learning module pipelines.

1. Dynamic Video Interpreter: The app accepts a YouTube link and utilizes Whisper AI to generate a highly accurate .srt transcript. I built a backend mechanism that processes this transcript word-by-word (including affixes) to query a dataset of SIBI (Indonesian Sign Language) video clips. These clips are then dynamically stitched and overlaid on the bottom right of the YouTube video, mimicking a live news broadcast interpreter.

2. Automated Assessment via Groq API: To eliminate manual work for teachers, I integrated Google Cloud Console with Firebase. Once the .srt is stored, a Cloud Function triggers the Groq API to analyze the transcript context and automatically generate multiple-choice questions (A-E) along with the correct answers, saving them directly to the database in real-time.

The X-Factor

Strict RAG Architecture against Hallucinations

While connecting the RAG pipeline to MongoDB Atlas was straightforward, the real challenge was deploying the model locally and ensuring zero hallucinations for academic use cases.

I utilized MongoDB Atlas's built-in Vector Search (using Cosine Similarity) to retrieve the top 6 most relevant document chunks from our localized database. The core engineering effort went into designing a highly restrictive prompting architecture for Llama 3.1.

The LLM was instructed via a strict scenario to only generate responses based on those 6 retrieved chunks. It was explicitly programmed to reject off-topic questions. Furthermore, if a query was on-topic but the exact answer wasn't present in the retrieved chunks, the system gracefully redirected the user to a human helpdesk, ensuring 100% information accuracy for university students.

Managing SDLC & Whitebox Testing Implementation

Leading the Software Engineering Project (PPL) for a location-based surplus food delivery app (similar to GoFood, utilizing Google Maps API) required balancing team dynamics with rigorous technical standards.

Beyond standard Project Manager duties—such as tracking weekly agile sprint targets via Trello and acting as the crucial communication bridge between the Frontend and Backend teams—the biggest technical challenge I tackled was Quality Assurance.

I initiated and oversaw the implementation of rigorous internal Whitebox Testing using Jest. This ensured that despite the rapid development cycle and integrating complex location-based APIs, the core backend logic and application state remained stable and resilient against regressions.
